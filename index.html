01 / Visão Geral

O Azure Health Suite é uma extensão para Google Chrome desenvolvida para equipes de desenvolvimento que utilizam o Azure DevOps. Ela gera dashboards de sprint, métricas de backlog e apresentações PPTX diretamente no navegador.

Esta política descreve de forma transparente quais dados a extensão acessa, onde eles ficam armazenados e como são utilizados. Em resumo: seus dados nunca saem do seu dispositivo para servidores de terceiros além do próprio Azure DevOps.

02 / Dados Acessados

Dado Finalidade Armazenamento
Personal Access Token (PAT) Autenticar chamadas à API do Azure DevOps local
Organização / Projeto / Team Identificar o workspace correto no Azure DevOps local
Work items (US, Bugs, Tasks) Exibir backlog, sprint e métricas no dashboard não salvo
Membros do time Calcular workload e recomendar analistas não salvo
Configurações de squad e skills Persistir preferências entre sessões local
Template do PPTX Gerar apresentações de Review local

local significa armazenado exclusivamente no chrome.storage.local do seu navegador, no seu dispositivo. não salvo significa que o dado é carregado em memória apenas durante a sessão e descartado ao fechar a aba.

04 / Permissões do Chrome

storage

Salvar configurações da squad, credenciais do Azure DevOps e preferências de template localmente no navegador.

tabs

Abrir o dashboard em uma nova aba ao clicar no ícone da extensão na barra do Chrome.

host_permissions: https://dev.azure.com/*

Realizar chamadas à REST API do Azure DevOps para buscar sprints, work items, membros do time e dados de backlog. Nenhuma outra origem é acessada.

05 / Segurança do Token (PAT)

Atenção
O Personal Access Token é armazenado em chrome.storage.local, que não é sincronizado com a nuvem. Recomendamos criar um PAT com escopo mínimo necessário (leitura de work items e membros do time) e com data de expiração definida.

O PAT é transmitido exclusivamente para https://dev.azure.com via HTTPS como cabeçalho de autorização HTTP Basic, conforme documentação oficial da Microsoft. Nenhuma cópia é enviada a qualquer outro destino.

06 / Dados de Terceiros

A extensão utiliza as seguintes bibliotecas de código aberto empacotadas localmente. Nenhuma delas realiza chamadas de rede durante o uso normal:

07 / Seus Direitos e Controle

Excluir todos os dados

Acesse chrome://extensions → Azure Health Suite → "Remover extensão". Todos os dados locais são apagados permanentemente.

Revogar acesso ao Azure DevOps

Acesse as configurações da sua conta Microsoft, seção "Personal Access Tokens", e revogue o token a qualquer momento.

Limpar configurações manualmente

Na aba de Configurações da extensão, utilize o botão "Limpar dados" para remover credenciais e preferências sem desinstalar.
